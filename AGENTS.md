## Learned User Preferences

- Always edit files directly when asked to make changes — do not just describe or propose changes
- Use BUN or PNPM (not npm) for package management, unless the user says "Im subbu :)"
- When the user says "refer @api.md", read the API documentation file before making any changes
- The user prefers sub/dub variants as separate server entries (e.g., "suzu-dub", "animepahe-dub") rather than in-player audio switching
- Prefer a strong blur overlay (around half strength) with centered "Coming Soon" text over removing or hiding unfinished features
- For API integration or device checks: curl expected responses per `api.md` first; when asked to test on device, install and verify on the connected Android device via adb
- BFF auth: in-app signup via `POST /v1/auth/signup`; logout invalidates the Anisurge JWT; no passwords in BFF/Postgres—mirror Project-R auth and link rows via `external_user_id` only; forward Project-R `error`/`message` on signup/login failures (e.g. 409)
- For Anisurge-owned backend data, prefer self-hosted **raw PostgreSQL** (not Mongo or Scylla for that layer); run Postgres **separately** from the API container so DB stays up when the API restarts
- Custom username and profile photo (`PATCH /v1/me`, `POST /v1/me/pfp/upload`) are **Anisurge-only** — do not sync to Project-R/ReAnime
- Playback preferences (auto-play, auto-next, skip intro/outro, default lang, sync %) must apply immediately via `SettingsStore` and account sync—not only after tapping Save in Settings; show intro/outro skip ranges on the player progress bar only (no Intro/Outro text labels beside timestamps); use source/batch captions only—**no in-app BFF AI subtitle translation** (reverted); planned: external VTT API + notify when translated subs are ready
- When a download fails, remove the failed queue entry and offer a fresh download—avoid resume/retry UX that stalls mid-job
- Berries and shop cosmetics are **cosmetic-only**—never gate streaming, server choice, or downloads
- **anisurge.lol releases:** one stable release per platform in admin (edit the current row, bump build); keep `GET /api/app/updates` JSON compatible with the shipped KMP client (`success`, `version`, `build`, `changelog`, `downloadUrl`, `updateAvailable`)

## Learned Workspace Facts

- KMP Compose app **anisuge** (`to.kuudere.anisuge`); targets Android + Compose Desktop (Linux/Windows/**macOS** DMG/portable zip); Ktor uses `ignoreUnknownKeys = true`; on Android, BFF JSON `PATCH`/`POST` must set `contentType(ContentType.Application.Json)` or body serialization fails
- Auth uses Bearer tokens in `SessionInfo` via `SessionStore`; BFF migration uses **dual tokens** — `project_r_…` for Project-R catalog/social and an **Anisurge JWT** for BFF routes (`/v1/auth/*`, `/v1/me`, watchlist, continue, progress)
- API hosts: anime/catalog/search/comments on `https://api.reanime.to/api/v1` (`AppComponent.BASE_URL`); home rows from `GET {BASE_URL}/home` (`latest_aired`, `new_on_site`, `upcoming`); latest paginated list `GET {BASE_URL}/home/latest-aired`; auth/library/profile on Anisurge BFF `https://db.anisurge.qzz.io` (`services/anisurge-api/api.md`); `STREAMING_URL` and `anisurge.lol` unchanged
- Streaming uses `AppComponent.STREAMING_URL` (`action=batch_scrape&anilistId=…`); Suzu needs proxy/player Referer; AnimePahe is usually direct; **Anitaku** returns vibeplayer `.m3u8` with `Referer: https://anitaku.online/` and per-quality `subtitles: [{ url, label }]` in batch_scrape — `BatchSubtitleExtract` for playback. Vibeplayer segments on `ibyteimg.com`/`byteimg.com` are PNG header + MPEG-TS after `IEND` — `HlsPngTsStrip` (common) strips for playback (`LocalStreamProxy` on ibyteimg/byteimg only; other proxied CDNs stream-through) and HLS downloads. Android Anitaku downloads: skip Media3 remote HLS export (`preferLocalTsRemux`); remux local `video_raw.ts` via RxFFmpeg first
- **anisurge.lol site** (`Anisurge-V2-site/`, Next.js, Mongo, Dragonfly): streaming server catalog; public downloads `GET /api/downloads` (android/windows/linux/macos, `?t=download` → `#download`); admin `/admin/releases` — one stable row per platform (PATCH by id, bump build). In-app `GET /api/app/updates` returns legacy KMP fields plus `files`; client sends `currentBuild`. Update cache must not treat cached `null` as final; pick enabled release else latest published (same as downloads). macOS file keys: `arm64-dmg`, `x64-dmg`, `arm64-zip`, `x64-zip`
- **Intro/outro skip** (`AniskipService`): `GET https://api.aniskip.com/v2/skip-times/{malId}/{episode}` with `types[]=op|ed` and `episodeLength`; resolve MAL from catalog `mal_id` or AniList GraphQL `idMal` (`MalAnilistIdCache`); fills `StreamingData.intro`/`outro` after stream load; auto-skip must react when timestamps arrive async
- Anisurge BFF (`services/anisurge-api/`, Bun+Hono+Drizzle, Postgres `external_user_id`, prod `https://db.anisurge.qzz.io`) vs Project-R/ScyllaDB at `api.reanime.to`. Optional **Upstash Redis** via `REDIS_URL` (`rediss://` TLS; `redis://` auto-upgrades for `upstash.io` hosts); unset = no HTTP response cache, in-memory chat rate limits only. When set: caches hot `GET /v1/me`, `/shop/items`, `/shop/owned`, `/watchlist`, `/watch/continue`, `/rewards/status` (short TTLs, `X-Cache` header); invalidates on writes; chat send uses Redis `SET NX PX 2000` per user; `GET /health/ready` reports `redis`. On login/sync upserts Project-R profile/settings, stores `project_r_token`; **two-way library merge** on login + `POST /v1/library/sync` (newer `lastUpdated` wins, push local-only rows to ReAnime). Watchlist folders uppercase: `WATCHING`, `PLANNING`, `COMPLETED`, `PAUSED`, `DROPPED`. Watchlist/continue writes still forward via `forwardToReanime` (default true). App calls library sync on pull-to-refresh watchlist and when loading continue watching (60s debounce). **Legacy sessions** (ReAnime token only, no `anisurgeToken`) are cleared on startup — no silent `/auth/sync`; user must log in again (TV QR pairing still uses `/auth/sync`). MAL/AniList tokens mirror via `GET/PATCH /v1/me/integrations`; on restore, if device has tokens but server row is empty, **push local once** instead of wiping
- Custom pfps + shop APNG frames: static pfps (`PFP_STORAGE_DIR`, `GET /v1/media/pfps/:filename`); shop frames (`PFP_ANIMATED_STORAGE_DIR`, `GET /v1/media/pfp-animated/{id}.png`); `/v1/shop/*` + `POST /v1/shop/redeem` (migration `0004_shop_pfp_animated` must be listed in `drizzle/meta/_journal.json`); unique `import_source` per APNG filename blocks duplicate catalog rows. Bulk upload script `shop:upload-one-by-one` dedupes via `GET /v1/admin/shop/import-sources` (not full `/v1/admin/shop/items`); `SHOP_ADMIN_SECRET` must match Coolify exactly. `DEFAULT_USER_COINS`, `SHOP_ADMIN_SECRET`; staff UI at `GET /admin` or `/staff` (users, redeem codes, shop; embedded HTML if `public/` missing); `SHOP_STAFF_EMAIL_ALLOWLIST` → shop tab only (not Users/Redeem); `PATCH /v1/admin/shop/items/:id`; optional shop AI (`SHOP_AI_*` env, multi-frame APNG for vision, min suggested price 20—never expose API key/base URL in staff UI). **Berries economy:** ledger `coin_transactions`; earn via `/v1/rewards/status` + `/rewards/daily-claim`, episode progress, first chat/day (env-capped); shop `price_coins` via BFF/admin only. App labels currency **Berries** (API field `coins`). Catbox not used for profile upload; render equipped frames with `apng-core` (~1.16× avatar, outer → PFP → frame)
- BFF Coolify deploy: Dockerfile pre-creates `/app/data/{pfps,pfp-animated,shop-videos}` owned by `anisurge`; set `ANISURGE_DATA_DIR=/app/data` and mount persistent storage at `/app/data` (e.g. host `/mnt/hdd/Anisurge/...` → container paths). **`DATABASE_URL` on Coolify must be the same Postgres** used for migrations/local shop work or catalog read/write diverges; optional `REDIS_URL` (`rediss://`) on the API service only
- **Community Chat** (global room; nav route still `live-chat`): BFF username snapshotted per message; avatar from live BFF profile (`customPfpUrl`); animated ring/outer from `equipped.chatAvatarFrame` / `chatAvatarOuter` (per-user equipped frame URLs — not a global test overlay). Chat images: `POST /v1/chat/upload` → Catbox URL in message (`imageUrl`)
- **Episode comments are Anisurge-owned (BFF), not ReAnime.** App `CommentService` → `/v1/comments/*` with the **Anisurge JWT** (`AnisurgeApi.applyAnisurgeAuth`): `GET /v1/comments/:animeId/:ep?page&sort&parent_id&nid` (root 20/page, replies 10/page, sort `new|oldest|best`), `POST /v1/comments` `{anime,ep,content,parentCommentId?,spoiler?}`, `POST /v1/comments/like` `{commentId,likeState:like|dislike|none}` (toggle-off on repeat), `DELETE /v1/comments/:id` (author or mod/admin, soft-delete). Each comment carries `authorFrameUrl`/`authorOuterUrl` from author `equipped` (chat parity) so `CommentsSection` renders equipped shop frames via `ProfileAvatar` + prefetches APNG bytes with `AnimatedFrameBytesCache`. **Community-post comments (`CommunityService`) stay on ReAnime** — only the watch-screen episode comments moved.
