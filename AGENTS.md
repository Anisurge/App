## Learned User Preferences

- Always edit files directly when asked to make changes — do not just describe or propose changes
- Use BUN or PNPM (not npm) for package management, unless the user says "Im subbu :)"
- When the user says "refer @api.md", read the API documentation file before making any changes
- The user prefers sub/dub variants as separate server entries (e.g., "suzu-dub", "animepahe-dub") rather than in-player audio switching
- Prefer a strong blur overlay (around half strength) with centered "Coming Soon" text over removing or hiding unfinished features
- For API integration or device checks: curl expected responses per `api.md` first; when asked to test on device, install and verify on the connected Android device via adb
- BFF auth: in-app signup via `POST /v1/auth/signup`; logout invalidates the Anisurge JWT; no passwords in BFF/Postgres—mirror Project-R auth and link rows via `external_user_id` only
- For Anisurge-owned backend data, prefer self-hosted **raw PostgreSQL** (not Mongo or Scylla for that layer); run Postgres **separately** from the API container so DB stays up when the API restarts
- Custom username and profile photo (`PATCH /v1/me`, `POST /v1/me/pfp/upload`) are **Anisurge-only** — do not sync to Project-R/ReAnime
- Playback preferences (auto-play, auto-next, skip intro/outro, default lang, sync %) must apply immediately via `SettingsStore` and account sync—not only after tapping Save in Settings
- Show intro/outro skip ranges on the player progress bar only; do not add Intro/Outro text labels beside the timestamps
- When a download fails, remove the failed queue entry and offer a fresh download—avoid resume/retry UX that stalls mid-job

## Learned Workspace Facts

- KMP Compose app **anisuge** (`to.kuudere.anisuge`); Ktor uses `ignoreUnknownKeys = true`; on Android, BFF JSON `PATCH`/`POST` must set `contentType(ContentType.Application.Json)` or body serialization fails
- Auth uses Bearer tokens in `SessionInfo` via `SessionStore`; BFF migration uses **dual tokens** — `project_r_…` for Project-R catalog/social and an **Anisurge JWT** for BFF routes (`/v1/auth/*`, `/v1/me`, watchlist, continue, progress)
- API hosts: anime/catalog/search/comments on `https://api.reanime.to/api/v1` (`AppComponent.BASE_URL`); home rows from `GET {BASE_URL}/home` (`latest_aired`, `new_on_site`, `upcoming`); latest paginated list `GET {BASE_URL}/home/latest-aired`; auth/library/profile on Anisurge BFF `https://db.anisurge.qzz.io` (`services/anisurge-api/api.md`); `STREAMING_URL` and `anisurge.lol` unchanged
- Streaming uses `AppComponent.STREAMING_URL` (`action=batch_scrape&anilistId=…`); Suzu needs proxy/player Referer; AnimePahe is usually direct; **Anitaku** returns vibeplayer `.m3u8` with `Referer: https://anitaku.online/` and per-quality `subtitles: [{ url, label }]` in batch_scrape — `BatchSubtitleExtract` for playback. Vibeplayer segments on `ibyteimg.com`/`byteimg.com` are PNG header + MPEG-TS after `IEND` — `HlsPngTsStrip` (common) strips for playback (`LocalStreamProxy` on ibyteimg/byteimg only; other proxied CDNs stream-through) and HLS downloads. Android Anitaku downloads: skip Media3 remote HLS export (`preferLocalTsRemux`); remux local `video_raw.ts` via RxFFmpeg first. Public site: streaming server catalog and app updates on `anisurge.lol`
- **Intro/outro skip** (`AniskipService`): `GET https://api.aniskip.com/v2/skip-times/{malId}/{episode}` with `types[]=op|ed` and `episodeLength`; resolve MAL from catalog `mal_id` or AniList GraphQL `idMal` (`MalAnilistIdCache`); fills `StreamingData.intro`/`outro` after stream load; auto-skip must react when timestamps arrive async
- Watchlist folder strings expected by API/UI are uppercase: `WATCHING`, `PLANNING`, `COMPLETED`, `PAUSED`, `DROPPED`
- Anisurge BFF (`services/anisurge-api/`, Bun+Hono+Drizzle, Postgres `external_user_id`, prod `https://db.anisurge.qzz.io`) vs Project-R/ScyllaDB at `api.reanime.to`. On login/sync upserts Project-R profile/settings, stores integration tokens, pulls watchlist/continue; writes default `forwardToReanime` to keep ReAnime in sync
- Custom pfps + shop APNG frames: static pfps (`PFP_STORAGE_DIR`, `GET /v1/media/pfps/:filename`); shop frames (`PFP_ANIMATED_STORAGE_DIR`, `GET /v1/media/pfp-animated/{id}.png`); `/v1/shop/*` (migration `0004_shop_pfp_animated` must be listed in `drizzle/meta/_journal.json`); `DEFAULT_USER_COINS`, `SHOP_ADMIN_SECRET`; admin catalog upload at `GET /admin/shop` (embedded HTML if `public/` missing); equip owned frames only on `PATCH /v1/me`. App shop UI labels currency **Berries** (API field `coins`). Catbox not used for profile upload
- BFF Coolify deploy: Dockerfile pre-creates `/app/data/{pfps,pfp-animated,shop-videos}` owned by `anisurge`; set `ANISURGE_DATA_DIR=/app/data` and mount persistent storage at `/app/data` (e.g. host `/mnt/hdd/Anisurge/...` → container paths)
- **Community Chat** (global room; nav route still `live-chat`): BFF username snapshotted per message; avatar from live BFF profile (`customPfpUrl`); animated ring/outer from `equipped.chatAvatarFrame` / `chatAvatarOuter` (per-user equipped frame URLs — not a global test overlay). Chat images: `POST /v1/chat/upload` → Catbox URL in message (`imageUrl`)
- APNG profile frames: use `io.github.lugf027:apng-core` on Android and desktop (Skia `Codec` only exposes one APNG frame); overlay ~1.16× avatar size in `ProfileAvatar`; draw order back→front: outer decoration, PFP, main frame on top (effects over the face)
- App code layout: services under `data/services/`, models under `data/models/`, screens/ViewModels under `screens/`; Navigation Compose path params (e.g. `info/{animeId}`). Settings **Frame shop** tab: avoid nested `LazyVerticalGrid` inside the settings scroll — use a width-based column grid; prefetch APNG bytes via `AnimatedFrameBytesCache` after catalog load
